Health Check

/health get

get

Prints information about the overall service health.

get /health

Prints information about the overall service health.

  • Response

HTTP status code 200

Body

Media type: application/json

Type: object

Properties
  • status: required(one of healthy, unhealthy)

    Status describes the overall service state. A status of "healthy" indicates that everything is running smoothly and all service dependencies are reachable and online. A status of "unhealthy" indicates that something is wrong with either this service or one or more of it's dependency services.

  • dependencies: required(array of DependencyStatus)

    A list of external services that this service depends on and some details about them.

    Items: DependencyStatus

    • name: required(string)

      Name of the external depdendency.

    • reachable: required(boolean)

      Whether or not the external service is reachable.

    • online: required(one of yes, unknown, no)

      Whether or not the external service is online and available for requests.

  • info: required(object)
    • threads: required(integer - minimum: 1)
    • uptime: required(string)
    • uptimeMillis: required(integer - minimum: 1)

HTTP status code 500

Body

Media type: application/json

Type: object

Properties
  • status: required(string)
  • message: required(string)
  • requestId: required(string)

User Dataset Uploading

/user-datasets get post

get

List user dataset uploads and their statuses

post

Prepare a new upload process.

Note: users are limited to one upload process at a time.

get /user-datasets

List user dataset uploads and their statuses

  • Response

HTTP status code 200

Body

Media type: application/json

Type: array of lib.StatusResponse

Items: StatusResponse

  • id: required(string - pattern: ^[a-f0-9]{8}-[a-f0-9]{4}-4[a-f0-9]{3}-[89aAbB][a-f0-9]{3}-[a-f0-9]{12}$)

    User dataset upload process id

  • name: required(string)

    User provided name for the upload

  • description: (string)

    Optional description of the user dataset.

  • projects: required(array of lib.ProjectType)

    Target projects for the dataset upload

  • started: required(datetime)

    Timestamp of the upload start

  • status: required(one of recieving-from-client, sending-to-handler, handler-unpacking, handler-processing, handler-packing, recieving-from-handler, sending-to-datastore, success, failed)

    Current job status, may be a completed status.

  • stepPercent: (integer - minimum: 0 - maximum: 100)

    Optional field avaible for process steps where the completion percent is calculable, such as during file transfer.

post /user-datasets

Prepare a new upload process.

Note: users are limited to one upload process at a time.

  • Request
  • Response

Body

Media type: application/json

Type: object

Properties
  • datasetName: required(string)

    Name for the user dataset. Will be visible in the site(s).

  • description: (string)

    Optional description for the user dataset.

  • projects: required(array of lib.ProjectType - minItems: 1)

    Target projects for which this user dataset should be made available for install.

  • summary: (string)

    Optional summary of the user dataset.
